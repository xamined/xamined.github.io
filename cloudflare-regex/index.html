Incident Timeline (UTC)

All timestamps from Cloudflare's official post-mortem published July 12, 2019.
13:42
New WAF rule deployed globally via Quicksilver. The rule targets XSS attacks and contains a regex with nested wildcards that triggers catastrophic backtracking.
13:42
CPUs across Cloudflare's edge network spike to 100%. The regex engine enters a backtracking loop on every incoming request. The rule was in simulate mode. It was only checking, not blocking.
13:42 - 13:45
Sites begin returning 502 and 503 errors. Discord, Shopify, and millions of other sites behind Cloudflare go down. The Cloudflare dashboard also goes down (it runs through their own network). Internal auth system Access also goes down. Some engineers have expired credentials.
13:45
PagerDuty fires the first alert. Engineers begin investigating but cannot access their own dashboard or admin tools.
14:07
An engineer executes a global WAF kill switch, disabling the entire Web Application Firewall.
14:09
Traffic begins recovering. CPU returns to normal levels.
